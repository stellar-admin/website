## Usage

The Input Tag Helper generates an [HTML Input Element](https://developer.mozilla.org/en-US/docs/Web/HTML/Reference/Elements/input) and uses the built-in [ASP.NET Core Input Tag Helper](https://learn.microsoft.com/en-us/aspnet/core/mvc/views/working-with-forms#the-input-tag-helper) internally.

```razor
<sa-input/>
```

All the same rules as the built-in Tag Helper applies for things like data binding and resolving input types from data annotation attributes. You can also use any of the [standard input attributes](https://developer.mozilla.org/en-US/docs/Web/HTML/Reference/Elements/input#attributes) and they will flow through correctly to the underlying `<input>` element.

### Implicit Field

You can also let StellarAdmin create an implicit field for an input.

<Demo src="/demo/tag-helpers/input-implicit-field.html">
  <include>./_include/input-implicit-field.mdx</include>
</Demo>

An implicit field will the created when you do either of the following:

1. Add the `label`, `description`, or `error` attributes.
1. Use data binding by specifying the `asp-for` attribute (see example below).

Even if either of the above is true, an implicit field will not be created under the following conditions:

1. The input is explicitly wrapped inside a Field Tag Helper
1. You opted out by setting the `render-field` attribute on the Input Tag Helper to `false`.

### Data binding

Bind the input to your model using the `asp-for` attribute. StellarAdmin will bind the value, set the `id` and `name` attributes, and display any validation errors.

<Demo src="/demo/tag-helpers/input-data-binding.html">

When using data binding, StellarAdmin will automatically wrap the input inside a [Field Tag Helper](field) and use data annotations for the label, description, placeholder, and validation messages. If you want to opt out of this behaviour, you can set the `render-field` attribute to `false`.

### Infer Input Types via Data Binding

When you use data binding, the input type will be inferred from the underlying data type and data annotations. For more information on this, refer to the built-in [ASP.NET Core Input Tag Helper documentation](https://learn.microsoft.com/en-us/aspnet/core/mvc/views/working-with-forms#the-input-tag-helper).
