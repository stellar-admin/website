## Usage

```razor
<dui-select>
    <option value="...">...</option>
</dui-select>
```

## Examples

### Model binding

The `dui-select` Tag Helper has full support for ASP.NET Core model binding using the `asp-for` and `asp-items` attributes. When using model binding, DuneUI will automatically wrap the select inside a [Field](field) with the correct label, description, and placeholder text derived from data attributes. You can opt out of this behavior by setting the `render-field` attribute to `false`

### Model binding validation

When using model binding, validation errors from [Model state](https://learn.microsoft.com/en-us/aspnet/core/mvc/models/validation) are displayed correctly.

<Demo src="/demo/tag-helpers/select-validation.html">
    <Tabs items={["Razor", "Model"]}>
        <Tab value="Razor">
        <include>./_include/select-validation.mdx</include>
        </Tab>
        <Tab value="Model">

        ```cs
        {
            [Display(
                Name = "Cabin Class",
                Description = "Higher classes offer more legroom, priority boarding, and flexible refund policies."
            )]
            [Required(ErrorMessage = "Please select a valid cabin class")]
            public CabinClass? CabinClass { get; set; }
        }
        ```
        </Tab>

</Demo>

### Manual Validation

If you're not using ASP.NET Core model binding, you can manually indicate validation errors by setting the `aria-invalid` attribute to `true`. This will display a red border around the select.

The error message can be displayed using either a `dui-field-error` Tag Helper (when using explicit fields) or the `error` attribute (when using implicit fields).
